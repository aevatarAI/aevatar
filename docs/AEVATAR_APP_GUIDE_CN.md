# AEVATAR App 市场同事使用手册（中文版）

面向对象：市场、售前、生态合作同事  
文档目标：让非研发同事可以在 10-15 分钟内完成一次完整的产品演示

---

## 1. 一句话理解产品

**AEVATAR App** 是一个可视化的 Workflow 操作台：  
你可以从“现成 Workflow 模板”开始，也可以用自然语言描述需求，自动生成并运行 Workflow，然后把结果沉淀为可复用的 YAML 流程资产。

---

## 2. 你能向客户展示什么

建议聚焦 4 个价值点：

1. **从想法到流程很快**：一句话就能生成可运行 Workflow 草稿。
2. **过程可解释**：图（Graph）、YAML、执行日志（Logs）三位一体，可追溯。
3. **支持人机协同**：流程可在关键节点等待人工输入、审批或外部信号。
4. **流程可沉淀**：验证通过后可保存为 YAML，形成企业流程资产库。

---

## 3. 演示前准备（3-5 分钟）

### 3.1 启动 App

```bash
aevatar app
```

常用变体：

```bash
# 指定端口，不自动打开浏览器
aevatar app --no-browser --port 6690

# 端口冲突/状态异常时重启
aevatar app restart
```

### 3.2 检查基础状态

- 页面左侧确认可看到 `LLM` 状态徽标
- 右上或侧边点击 `Config` 按钮可进入配置页（用于补齐 LLM/配置）
- 至少能看到一组 `Workflow Library` 条目

---

## 4. 页面导览（英文界面中文速记）

| 界面入口 | 你可以这样解释 |
|---|---|
| `Overview` | 产品总览：看库分组、能力分布、推荐流程 |
| `Workflows` | 模板库：选择现成 Workflow 直接运行 |
| `YAML` | YAML 浏览器：对比库流程和草稿流程 |
| `Primitives` | 原子能力库：每个步骤类型的说明与示例 |
| `Playground` | 创作区：用自然语言生成草稿、调试、运行、保存 |

---

## 5. 三条标准演示路径（建议按顺序）

## 路径 A：模板即开即用（2-3 分钟）

适用场景：首次介绍产品，强调“上手快”。

1. 进入 `Workflows`，从左侧库选择一个模板。
2. 在详情页展示三块内容：`Workflow Graph`、`Workflow YAML Snapshot`、`What This Workflow Does`。
3. 在 `Run This Workflow` 输入框输入一句业务请求，点击 `Run`。
4. 演示执行过程：
   - `Execution Log`：看每一步发生了什么
   - `Result`：看最终输出
5. 用一句话收尾：**“模板开箱即用，适合快速 PoC 和业务试跑。”**

## 路径 B：自然语言生成 Workflow（5 分钟）

适用场景：强调“低门槛 + 高效率”。

1. 进入 `Playground`。
2. 在底部输入框输入需求，例如：  
   “请生成一个客户咨询分流流程：先分类，再决定自动回复或转人工，并记录结果。”
3. 点击 `Send`，等待系统输出草稿（含 YAML 片段）。
4. 展示系统自动渲染：
   - 左侧 `Workflow YAML`
   - 右侧 `Workflow Graph`
5. 点击 `Run Draft` 执行草稿，观察 `Run Controls` 与 `Logs/Result`。
6. 若结果符合预期，点击 `Save YAML` 保存草稿。

注意：

- 只有**AI 生成且校验通过**的 YAML 才能保存。
- 保存成功后，该流程会进入库，可用于后续复用与分享。

## 路径 C：展示“人机协同节点”（3-5 分钟）

适用场景：强调“可治理、可控”。

推荐示例 Workflow（库内常见）：

- `human_input_basic_auto_resume`
- `human_approval_approved_auto_resume`
- `wait_signal_manual_success`
- `mixed_human_approval_wait_signal`

演示步骤：

1. 选择包含 `human_input / human_approval / wait_signal` 的 Workflow。
2. 点击 `Run` 后，演示流程在 `Human Interaction` 处暂停。
3. 输入审批/回复后继续运行，观察结果收敛。

一句话价值表达：  
**“AEVATAR 不是黑盒自动化，而是可在关键节点插入人工决策的协同自动化。”**

## 8. 常见问题与处理

### Q1：`Run` 按钮是灰的，点不了？

通常是当前草稿还不可运行：

- 在 `Playground` 里需要先生成并通过 YAML 校验
- 或者流程步骤为空/解析失败  

建议操作：重新发送需求描述，确认图和 YAML 已正常出现。

### Q2：`Save YAML` 按钮不可用？

常见原因：

- 当前 YAML 不是 AI 生成内容
- YAML 还未通过解析/校验
- 当前仍在运行中

建议操作：先完成一次成功生成并校验，再保存。

### Q3：演示时流程卡在等待状态？

如果是 `human_input / human_approval / wait_signal`，这是预期行为，表示进入“人工/信号关口”。  
在 `Interaction` 面板补充输入或审批即可继续。

### Q4：页面状态混乱，如何快速恢复？

优先顺序：

1. 点当前页面的 `Reset`（若可见）
2. 重新选择 Workflow
3. 执行 `aevatar app restart` 重启 App

## 9. 安全与演示边界建议

- 演示环境不要输入真实客户隐私数据
- 使用脱敏或模拟数据进行展示
- 对外演示时，重点讲“能力和流程”，少暴露内部配置细节
